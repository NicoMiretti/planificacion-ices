# Stakeholders

| Stakeholder | Rol / Interés | Influencia | Necesidad principal |
|---|---|---|---|
| Profesor | Usuario primario | Alta | Subir y reenviar planificaciones de forma simple, ver feedback claro. |
| Moderador / Coordinador | Usuario primario | Alta | Revisar de forma ordenada, dejar observaciones, llevar control mensual. |
| Autoridades de la facultad (Decano, Sec. Académica) | Sponsor | Media-Alta | Trazabilidad, reportes, cumplimiento. |
| Secretaría académica | Apoyo administrativo | Media | Acceso a planificaciones oficiales archivadas. |
| Departamento de sistemas / IT | Soporte técnico | Media | Mantenibilidad, seguridad, integración. |
| Alumnos | Beneficiario indirecto | Baja | (No usuario directo) Que la planificación oficial sea pública/consultable — confirmar. |

> Validar lista y agregar nombres concretos en la entrevista.
